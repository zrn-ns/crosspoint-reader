# 空ディレクトリ自動削除 設計書

Issue: https://github.com/zrn-ns/crosspoint-jp/issues/33

## 概要

ファイル削除や移動の結果として残った空のディレクトリを、ホーム画面に戻ったタイミングで自動的に削除する。

## トリガー

ホーム画面（FileBrowserActivity）の `onEnter()` 時に実行。
リアルタイム削除ではなく、ホーム画面遷移時のクリーンアップとすることで、ファイル整理中にディレクトリが勝手に消える問題を回避する。

## 対象スコープ

SDカードルート直下のユーザーディレクトリのみを再帰スキャン。

**除外対象:**
- `.` で始まるディレクトリ（`.crosspoint`, `.sleep` 等）
- システムディレクトリ（`System Volume Information`, `XTCache`）

## アルゴリズム

1. ルート `/` を開き、直下のディレクトリを列挙
2. 保護対象（`.` 始まり、システムディレクトリ）を除外
3. 各ディレクトリに対して再帰的に空チェック:
   - サブディレクトリがあればまず再帰処理
   - `openNextFile()` でエントリが0件なら `rmdir()` で削除
4. リーフ（末端）から順に削除するため、`/Aozora/太宰治/` → `/Aozora/` のように連鎖的に空ディレクトリが削除される

## パフォーマンス

- ディレクトリエントリの列挙のみでファイル内容は読まない
- `yield()` + `esp_task_wdt_reset()` でウォッチドッグ対策
- ファイル一覧表示（`loadFiles()`）の前に実行するため、削除済みディレクトリは表示されない

## ファイル構成の変更

| ファイル | 変更種別 | 内容 |
|----------|----------|------|
| `lib/hal/HalStorage.h` | 変更 | `removeEmptyDirs()` メソッド宣言追加 |
| `lib/hal/HalStorage.cpp` | 変更 | `removeEmptyDirs()` 実装（再帰スキャン＋削除） |
| `src/activities/home/FileBrowserActivity.cpp` | 変更 | `onEnter()` にクリーンアップ呼び出し追加 |

## Done 判定基準

- [ ] `pio run` でビルドエラー・警告なし
- [ ] 空ディレクトリが存在する状態でホーム画面に遷移すると削除される
- [ ] `.crosspoint` 等の保護ディレクトリは削除されない
- [ ] ネストした空ディレクトリ（例: `/Aozora/太宰治/`）が連鎖的に削除される
- [ ] 空でないディレクトリは削除されない
- [ ] 既存機能（ファイル一覧表示等）が壊れていない
※ 必須ゲート（動作検証・既存機能・差分確認・シークレット）は常に適用
